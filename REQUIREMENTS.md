# 要件定義書

## 1. システム概要

本システムは、被写体側 iOS アプリと撮影者側 Web アプリを用いて、参照写真から生成したガイドを見ながら協調的に写真撮影を行うためのシステムである。

被写体側 iOS アプリは、参照写真の選択、ガイド生成、ガイド保存、撮影セッション作成、ライブ映像の受信、撮影者画面へ表示するガイドの共有、受信写真の確認と保存を行う。

撮影者側 Web アプリは、スマートフォンのカメラ映像上にガイドを重ねて表示し、写真撮影、ライブ映像共有、写真送信、セッション保存データ削除を行う。

backend は、静的ファイル配信、アップロードファイル管理、ガイド生成、WebSocket 中継、写真・ガイドの API 提供を行う。

## 2. 利用者

- 被写体: iOS アプリを操作する利用者
- 撮影者: Web アプリを操作し、写真を撮影する利用者

## 3. 対象プラットフォーム

- 被写体側: iOS アプリ
- 撮影者側: Web ブラウザ
- サーバー側: Node.js backend
- ガイド生成処理: Python

## 4. 画面構成

### 4.1 iOS アプリ

iOS アプリは以下のタブを持つ。

- ホーム
- 写真
- ガイド作成
- ガイド表示
- 受信写真

### 4.2 撮影者 Web

撮影者 Web は以下の UI を持つ。

- カメラ映像表示
- ガイド画像オーバーレイ
- 接続状態表示
- ガイド ON/OFF ボタン
- ガイド透過度スライダー
- 撮影ボタン
- 撮影枚数表示
- ライブ共有ボタン
- 送信ボタン
- クリアボタン
- 削除ボタン
- 撮影サポート表示領域
- エラー表示領域
- 撮影済みサムネイル一覧

## 5. 機能要件

### 5.1 ガイド作成

- iOS アプリは、写真ライブラリから参照写真を選択できる。
- iOS アプリは、参照写真に対して 3:4 のトリミング枠を表示できる。
- iOS アプリは、トリミング枠の移動、拡大、縮小、リセットを行える。
- iOS アプリは、backend に参照写真とトリミング情報を送信し、ガイドを生成できる。
- backend は、同じ参照写真と同じトリミング情報から以下の3種類のガイドを生成できる。
  - 人物枠
  - 人物枠 + 主要ポイント
  - シルエット
- iOS アプリは、生成された3種類のガイドを切り替えてプレビューできる。
- iOS アプリは、ガイドの左右位置と大きさを調整できる。
- iOS アプリは、調整後のガイドセットを保存できる。

### 5.2 ガイド保存・管理

- iOS アプリは、保存済みガイドを一覧表示できる。
- 保存済みガイドは、参照写真と生成ガイド画像を持つ。
- 新しく保存されるガイドは、3種類のガイド画像を持つ。
- 既存の単一ガイド画像も表示できる。
- iOS アプリは、保存済みガイドの詳細を表示できる。
- iOS アプリは、保存済みガイドを削除できる。
- iOS アプリは、複数の保存済みガイドを選択して撮影画面へ渡せる。

### 5.3 撮影セッション

- iOS アプリは、撮影開始時にセッション ID を生成できる。
- iOS アプリは、撮影者 Web 用 URL を QR コードとして表示できる。
- 撮影者 Web は、URL の `sessionId` クエリからセッション ID を取得する。
- 撮影者 Web は、セッション ID がない場合にエラーを表示する。

### 5.4 WebSocket 接続

- backend は `/ws/video` を提供し、撮影者 Web から被写体 iOS アプリへライブ映像フレームを中継する。
- backend は `/ws/session` を提供し、ガイド更新、ガイド調整、セッション削除を撮影者 Web へ通知する。
- iOS アプリは、WebSocket 接続状態として「接続中」「接続済み」「切断」「再接続中」を表示できる。
- 撮影者 Web は、接続状態として「接続中」「接続済み」「切断」「再接続中」を表示できる。
- iOS アプリは、接続が切れた場合に再接続を試みる。
- 撮影者 Web は、セッション通知用 WebSocket が切れた場合に再接続を試みる。

### 5.5 ライブ映像共有

- 撮影者 Web は、ブラウザのカメラを起動できる。
- 撮影者 Web は、ライブ共有開始時にカメラ映像を JPEG フレームとして WebSocket 送信できる。
- iOS アプリは、撮影者 Web から受信したライブ映像を表示できる。
- 撮影者 Web は、ライブ共有を停止できる。
- 撮影者 Web は、カメラ起動失敗やライブ共有失敗をエラー表示できる。

### 5.6 ガイド共有・表示

- iOS アプリは、撮影者 Web へ表示するガイド画像を backend にアップロードできる。
- backend は、アップロードされたガイド画像を保存し、撮影者 Web にガイド更新を通知できる。
- 撮影者 Web は、通知されたガイド画像をカメラ映像上に重ねて表示できる。
- iOS アプリは、自分の画面に表示するガイド種類を選択できる。
- iOS アプリは、撮影者 Web へ送るガイド種類を選択できる。
- 自分の画面に表示するガイド種類の変更は、撮影者 Web へ送るガイドには影響しない。
- 撮影者 Web へ送るガイド種類を変更した場合、iOS アプリは選択中のガイド画像を再アップロードできる。
- iOS アプリは、撮影者 Web に表示されるガイドの透明度、左右位置、大きさを調整できる。
- 撮影者 Web は、受信したガイド調整をリアルタイムに反映できる。
- 撮影者 Web は、ガイド調整通知を受け取ったとき「ガイドが調整されました。」を撮影サポート領域に表示できる。
- 撮影者 Web は、ガイド表示の ON/OFF を切り替えられる。
- 撮影者 Web は、ガイド透過度をローカルに変更できる。

### 5.7 写真撮影・送信

- 撮影者 Web は、カメラ映像から写真を撮影できる。
- 撮影者 Web は、撮影済み写真のサムネイルを表示できる。
- 撮影者 Web は、撮影済み写真を個別に削除できる。
- 撮影者 Web は、撮影済み写真をすべてクリアできる。
- 撮影者 Web は、撮影済み写真を backend に送信できる。
- backend は、受信した写真をセッション単位で保存できる。
- iOS アプリは、セッションに保存された写真一覧を取得できる。
- iOS アプリは、受信写真を一覧表示できる。
- iOS アプリは、受信写真を1枚ずつ写真ライブラリへ保存できる。
- iOS アプリは、受信写真をまとめて写真ライブラリへ保存できる。

### 5.8 セッションデータ削除

- 撮影者 Web は、現在のセッションのサーバー保存データを削除できる。
- backend は、`DELETE /api/session/:sessionId` によりセッションの保存ファイルを削除できる。
- backend は、セッションデータ削除を撮影者 Web へ通知できる。
- backend は、一定時間を過ぎたセッション保存データを自動削除できる。

### 5.9 エラー表示

- 撮影者 Web は、以下の場合にエラーを表示できる。
  - セッション ID がない
  - サーバーに接続できない
  - WebSocket 接続が切断された
  - カメラを起動できない
  - カメラ映像が撮影可能な状態ではない
  - ライブ映像送信に失敗した
  - ガイド画像取得に失敗した
  - 写真送信に失敗した
  - セッションデータ削除に失敗した
- iOS アプリは、以下の場合にメッセージまたはエラーを表示できる。
  - ガイド生成に失敗した
  - ガイド共有に失敗した
  - 受信写真取得に失敗した
  - 写真ライブラリへの保存に失敗した
  - 写真ライブラリへのアクセスが許可されていない

## 6. API 要件

backend は以下の HTTP API を提供する。

- `GET /`
  - 撮影者 Web の `index.html` を返す。
- `GET /health`
  - サーバー起動確認用 JSON を返す。
- `POST /api/photos`
  - セッション ID と写真ファイルを受け取り、保存する。
- `GET /api/photos?sessionId=...`
  - 指定セッションの保存済み写真一覧を返す。
- `POST /api/session/:sessionId/guide`
  - 指定セッションのガイド画像を保存する。
- `GET /api/session/:sessionId/guide`
  - 指定セッションの最新ガイド画像を返す。
- `POST /api/session/:sessionId/generate-guide`
  - 指定した1種類のガイドを生成する。
- `POST /api/session/:sessionId/generate-guide-set`
  - 3種類のガイドセットを生成する。
- `DELETE /api/session/:sessionId`
  - 指定セッションの保存データを削除する。

backend は以下の WebSocket エンドポイントを提供する。

- `/ws/video`
  - `sessionId` と `role` をクエリで受け取る。
  - `role=sender` は撮影者 Web からの映像フレーム送信側である。
  - `role=receiver` は iOS アプリの映像受信側である。
- `/ws/session`
  - `sessionId` をクエリで受け取る。
  - ガイド更新、ガイド調整、セッション削除を通知する。

## 7. データ保存要件

### 7.1 iOS アプリ内保存

iOS アプリは、保存済みガイド情報を Documents ディレクトリ内の `guides.json` に保存する。

iOS アプリは、参照画像とガイド画像を Documents ディレクトリに保存する。

### 7.2 backend 保存

backend は、アップロードされた写真、参照画像、ガイド画像を `backend/uploads/<sessionId>/` に保存する。

backend は、セッション ID に含まれる英数字、ドット、アンダースコア、ハイフン以外の文字をアンダースコアに置換する。

## 8. 設定要件

- backend のポートは `PORT` 環境変数で変更できる。
- backend のセッション保存データ自動削除時間は `SESSION_TTL_MS` 環境変数で変更できる。
- backend の自動削除確認間隔は `CLEANUP_INTERVAL_MS` 環境変数で変更できる。
- backend が使用する Python コマンドは `PYTHON` 環境変数で変更できる。
- iOS アプリのサーバー URL は `Info.plist` の `ServerBaseURL` で設定できる。
- iOS アプリの WebSocket URL は `Info.plist` の `WebSocketBaseURL` で設定できる。

## 9. 使用技術

- iOS アプリ: SwiftUI, UIKit, PhotosUI, Photos, CoreImage, URLSession, URLSessionWebSocketTask
- 撮影者 Web: HTML, CSS, JavaScript, MediaDevices API, Canvas API, WebSocket, Fetch API
- backend: Node.js, Express, Multer, ws, cors
- ガイド生成: Python, Pillow, OpenCV, NumPy, Ultralytics

## 10. 制約・前提

- 撮影者 Web でカメラを使用するため、ブラウザのカメラ許可が必要である。
- スマートフォン実機で利用する場合、iOS アプリと撮影者 Web の両方から backend に到達できる必要がある。
- ガイド生成処理は backend から Python スクリプトを実行して行う。
- 撮影者 Web から送信されるライブ映像は、WebSocket 経由の JPEG フレームとして iOS アプリへ中継される。
